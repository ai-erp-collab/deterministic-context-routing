# Practical Adoption Checklist

## Goal

Застосувати методологію до одного модуля, не намагаючись спершу задокументувати всю систему.

## 1. Select The Module

- [ ] Назвіть активний модуль.
- [ ] Підтвердіть його шлях у `modules.md`.
- [ ] Підтвердіть корені wiki, джерел і артефактів.
- [ ] Прочитайте `AGENTS.md` модуля.
- [ ] Прочитайте лише останній запис `session_state.md` модуля.

## 2. Record The Capability Profile

- [ ] Зазначте, чи доступний Git; якщо так, назвіть дозволені команди статусу, diff та історії.
- [ ] Зазначте, чи доступна верифікація компілятора/збірки; якщо так, назвіть точну команду.
- [ ] Зазначте, чи доступні автоматизовані тести; якщо так, назвіть точну команду та надійний обсяг.
- [ ] Зазначте, чи доступні локальні чи цільові runtime-перевірки.
- [ ] Зазначте потрібні статичні чи захисні перевірки.
- [ ] Тримайте оновлення стану сесії обов'язковими, бо вони фіксують намір завдання, свідчення, припущення, верифікацію та наступну дію.

## 3. Write The Module Concept

- [ ] Зазначте бізнес-призначення модуля.
- [ ] Визначте основні бізнес-терміни.
- [ ] Назвіть таблиці, документи, API та workflow, якими володіє модуль.
- [ ] Назвіть сусідні модулі та межі.
- [ ] Явно зафіксуйте невідомі.
- [ ] Позначте важливі відображення як `confirmed`, `inferred`, `runtime-unverified` чи `blocked-by-access`.

## 4. Curate One Evidence Artifact

- [ ] Розмістіть нові сирі експорти в налаштованій папці сирих артефактів.
- [ ] Визначте, чи є артефакт схемою таблиці, прикладом джерела, документом користувача, runtime-нотаткою чи бізнес-правилом.
- [ ] Створіть чи оновіть найменшу корисну сторінку wiki.
- [ ] Додайте мітки впевненості до важливих тверджень.
- [ ] Переміщуйте оброблені експорти таблиць/джерел лише після того, як сторінку wiki створено чи оновлено.
- [ ] Оновіть індекс артефактів.

## 5. Run One Workflow-Backed Task

- [ ] Завантажте лише концепцію модуля та найменші релевантні сторінки wiki.
- [ ] Напишіть короткий план завдання перед реалізацією.
- [ ] Зафіксуйте припущення, які не можна перевірити локально.
- [ ] Внесіть найменшу корисну зміну.
- [ ] Виконайте наявну перевірку Git, перевірки компілятора/збірки, автоматизовані тести, runtime-перевірки, статичні перевірки та захисні сканування згідно з профілем можливостей.

## 6. Record The Result

- [ ] Оновіть `session_state.md` модуля.
- [ ] Оновіть кореневий `session_state.md` як компактний індекс активності модуля.
- [ ] Додайте довготривалі відкриття до wiki модуля чи спільної wiki.
- [ ] Додайте повторювані проблеми workspace до `known_issues.md`.

## Done Criteria

- Наступний агент може визначити модуль, стан завдання, використані свідчення, виконану верифікацію та залишкові припущення без читання історії чату.
- Принаймні одна курована сторінка чи довготривала нотатка існує завдяки завданню.
- Метод доведено на одному вузькому зрізі перед ширшим впровадженням.
